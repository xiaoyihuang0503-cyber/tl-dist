# 物品数据库冷备档

每赛季开始时冻结一份 `item_db.json`（物品 ID → 中文名）和 `item_img_urls.json`（物品 ID → 图标 URL），防止上游数据源消失。

## 文件命名

`v<游戏版本>-<冻结日期>.json`

例如 `v1.3.0-2026-04-14.json` 表示 1.3.0 赛季开赛时冻结。

## 历史快照

| 赛季 | 游戏版本 | 冻结日期 | 物品条目 | 文件 |
|------|---------|---------|---------|------|
| 1.3.0 | 1.3.0 | 2026-04-14 | ~ | [v1.3.0-2026-04-14.json](v1.3.0-2026-04-14.json) |

## 使用方式

客户端打包时从此目录取最新版本嵌入 `dist/`。新赛季上线后先从游戏或 ETor 侧 dump 新版，审查增量条目，命名新文件追加到此目录。
